# カラーミーショップ本番 CSV 取得手順

> **目的:** 本番移行用の CSV をカラーミー管理画面から取得し、リポジトリの `colorme_data/` に配置する。  
> **参照:** [データベース仕様書 §6](./specification.md#6-カラーミー-csv-との対応)、[テーブル定義書](./table-definition.md)

---

## 1. 概要

| 取得する CSV | カラーミーのデータ種類 | ダウンロード時のファイル名 | 配置先（リネーム後） |
|-------------|----------------------|---------------------------|---------------------|
| 商品 | 商品データ | `product.csv` | `colorme_data/product.csv` |
| オプション | 商品オプションデータ | `boption_csv_download_YYYYMMDD_HHMMSS.csv` 等 | `colorme_data/option.csv`（推奨） |
| 顧客 | 顧客データ | `customer.csv` | `colorme_data/customer.csv` |
| 注文 | 受注一括データ | `sales_all.csv` | `colorme_data/sales_all.csv` |

インポートコマンド（予定）は上記配置先を参照する。オプション CSV だけはダウンロード時のファイル名が可変のため、**`option.csv` にリネーム**しておく。

---

## 2. 事前準備

### 2.1 権限

- カラーミーショップの **管理者アカウント** でログインできること
- **データダウンロード** 機能が利用可能なプランであること

### 2.2 ダウンロード用パスワード（初回のみ）

ショップによっては、データダウンロードの前に **ダウンロード用パスワード** の設定が必要です。

1. 管理画面 → **アカウント・設定** → **オーナー情報編集**
2. **その他パスワード設定**（ダウンロード用パスワード）を設定して保存
3. 以降、データダウンロード画面でこのパスワードを入力

※ メニュー名はカラーミーの UI 更新で変わる場合があります。見つからない場合は [公式ヘルプ：データダウンロード](https://help.shop-pro.jp/hc/ja/articles/360062479334) を参照。

### 2.3 取得タイミング（本番移行）

| タイミング | 推奨 |
|-----------|------|
| **リハーサル移行**（ステージング） | 本番切替の **1〜2 週間前** に取得。以降の新規受注は移行対象外になるため、リハーサル後に再取得が必要 |
| **本番移行** | **サイト公開・カラーミー閉店の直前**（受注停止後）に最終版を取得 |
| **商品・顧客** | 注文より頻繁に変わらないが、本番移行直前に **再ダウンロード** して最新にする |

## 3. 共通のダウンロード手順

### 3.1 画面の開き方

**最新 UI（2024 年以降）**

1. カラーミー管理画面にログイン
2. 左サイドバー下部の **「ダウンロード」** をクリック

**旧 UI の場合**

1. **商品管理** → **データ管理**（または **データダウンロード**）

### 3.2 共通設定

| 項目 | 推奨値 | 理由 |
|------|--------|------|
| **項目名 表示設定** | **表示する** | 1 行目にヘッダーが付き、インポータが列名でマッピングする |
| **対象データ** | データ種類ごとに下記 §4 参照 | — |

### 3.3 ダウンロード後の共通作業

1. **文字コード:** カラーミーの CSV は **Shift_JIS（CP932）** のことが多い。Excel で開いて上書き保存すると文字化け・列ずれの原因になる。**テキストエディタやインポートコマンド側で変換**する（`CsvReader` で処理予定）。
2. **右端の空列:** 商品系 CSV には末尾に空列が付く場合がある。インポート前に列数を確認する（不要な空列はインポータ側で無視する想定）。
3. **`colorme_data/` へ配置:** 下表のリネーム後ファイル名で保存する。
4. **Git にコミットしない:** 顧客・注文 CSV には個人情報が含まれる。`.gitignore` で `colorme_data/*.csv` を除外済み。

```
iorishobo/
└── colorme_data/
    ├── product.csv
    ├── option.csv          ← オプション（リネーム後）
    ├── customer.csv
    └── sales_all.csv
```

## 4. データ種類ごとの手順

### 4.1 商品データ（`product.csv`）

| 項目 | 内容 |
|------|------|
| データ種類 | **商品データ** |
| ファイル名 | `product.csv` |
| 想定件数 | **約 72 行**（親商品数。本番でも同程度） |
| 移行先 | `categories`, `products`, `product_images`（画像 URL は別途ダウンロード） |

**手順**

1. ダウンロード画面で **データ種類 → 商品データ** を選択
2. **項目名 表示設定 → 表示する**
3. **ダウンロード** をクリック
4. `colorme_data/product.csv` として保存

**移行で使う主な列（参考）**

商品 ID、大/小カテゴリー、商品名、商品画像 URL、その他画像 1〜9 URL、販売価格、在庫数、在庫管理、掲載設定、簡易説明、商品説明 など。  
定価・型番・ISBN・重量・販売期間等は [仕様書](./specification.md) のとおり **スキップ**。

---

### 4.2 商品オプションデータ（`option.csv`）

| 項目 | 内容 |
|------|------|
| データ種類 | **商品オプションデータ** |
| ファイル名 | `boption_csv_download_YYYYMMDD_HHMMSS.csv`（タイムスタンプ付き） |
| 移行先 | `product_variants` |

**手順**

1. **データ種類 → 商品オプションデータ** を選択
2. **項目名 表示設定 → 表示する**
3. **ダウンロード** をクリック
4. **`colorme_data/option.csv` にリネーム**して保存（最新 1 ファイルのみ残す）

**移行で使う主な列**

商品 ID、オプション名 1/2、オプション ID、在庫数、販売価格 など。型番はスキップ。

---

### 4.3 顧客データ（`customer.csv`）

| 項目 | 内容 |
|------|------|
| データ種類 | **顧客データ** |
| ファイル名 | `customer.csv` |
| 想定件数 | **数千件**（本番） |
| 移行先 | `customers`, `users`（会員のみ） |

**手順**

1. **データ種類 → 顧客データ** を選択
2. **項目名 表示設定 → 表示する**
3. **ダウンロード** をクリック
4. `colorme_data/customer.csv` として保存

**注意**

- **「ユーザー登録」列**（有/無）が含まれること。会員判定に使用（[仕様書 §3.20](./specification.md#320-顧客会員カラーミー型)）
- パスワード列は **含まれない**（移行会員はランダムパスワード + 再設定フロー）
- FAX 列は移行対象外

---

### 4.4 受注一括データ（`sales_all.csv`）

| 項目 | 内容 |
|------|------|
| データ種類 | **受注一括データ** |
| ファイル名 | `sales_all.csv` |
| 想定件数 | **数千件**（本番） |
| 移行先 | `orders`, `order_items` |

**手順**

1. **データ種類 → 受注一括データ** を選択
2. **項目名 表示設定 → 表示する**
3. **対象データ** を指定（下記）
4. **データ出力条件** は **すべての注文を含める** よう、フィルタを外す  
   - 「発送済・受注キャンセル済のものを除く」等のチェックは **外す**（過去の発送済・キャンセルも移行する）
5. **ダウンロード** をクリック
6. `colorme_data/sales_all.csv` として保存（分割取得時は §4.4.1 参照）

#### 4.4.1 全期間を取得する場合（数千件・長期運用）

受注一括データは **期間指定で一度に取れるのは最大 180 日** です（[公式ヘルプ](https://help.shop-pro.jp/hc/ja/articles/360062479334)）。  
それより古いデータは **年月指定** で取得します。

**推奨フロー**

```
1. 年月指定でショップ開設月から順にダウンロード（例: 2020-01, 2020-02, …）
2. 直近 180 日以内は期間指定でも可
3. 各 CSV のヘッダー行を除き、データ行だけを 1 つの sales_all.csv に結合
   ※ ヘッダーは先頭ファイルの 1 行だけ残す
4. 売上 ID の重複がないか確認（結合ミスの検証）
```

**結合時の確認**

| チェック項目 | 目安 |
|-------------|------|
| 売上 ID の重複 | 0 件であること |
| 行数 | 管理画面の累計受注数とおおむね一致 |
| 必須列 | 売上 ID、購入者 名前、購入者 メールアドレス、商品明細 等が空でないこと |

**移行で使う主な列（参考）**

売上 ID、受注日、PC・携帯区分、購入者・配送先各列、金額列、決済方法、入金状態、発送状態、購入商品関連列 など。  
ギフト・のし・ラッピング関連列はスキップ（[仕様書 §3.16](./specification.md#316-ギフトのしラッピング等)）。

## 5. 取得後のチェックリスト

移行コマンド実行前に、以下を確認する。

- [ ] 4 種類の CSV が `colorme_data/` に揃っている
- [ ] オプション CSV を `option.csv` にリネームした
- [ ] 各ファイルの 1 行目に **項目名（ヘッダー）** がある
- [ ] `product.csv` の行数が **おおよそ 72 前後** である
- [ ] `customer.csv` / `sales_all.csv` の件数が本番想定（数千件）と一致する
- [ ] `sales_all.csv` で売上 ID に **重複がない**
- [ ] CSV を Excel で編集していない（した場合は再ダウンロードを推奨）
- [ ] ファイルを Git にコミットしていない

## 6. インポート順序（参考）

CSV 配置後の移行コマンド実行順序（[tasks.md](./tasks.md) より）:

```
# 一括実行（推奨）
php artisan import:colorme-all

# 個別実行
1. php artisan import:colorme-products    … product.csv + option.csv
2. php artisan download:product-images    … 商品画像 URL からローカル保存
3. php artisan import:colorme-customers   … customer.csv
4. php artisan import:colorme-orders      … sales_all.csv（顧客・商品の後）
```

`import:colorme-all` は上記を順番に実行し、件数突合・slug 衝突・移行ログ（SKIP/ERROR）のレポートを表示する。画像ダウンロードを省略する場合は `--skip-images` を付ける。インポート済み DB の検証のみは `--verify-only`。
